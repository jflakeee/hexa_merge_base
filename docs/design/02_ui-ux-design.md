## 1. 화면 구성 (Screen Layout)

게임은 총 6개의 주요 화면으로 구성된다. 각 화면은 명확한 계층 구조와 일관된 디자인 언어를 유지하며, 최소한의 탭으로 원하는 기능에 도달할 수 있도록 설계한다.

### 1.1 화면 계층 구조

```
메인 메뉴 (Main Menu)
├── 게임 플레이 (Gameplay)
│   ├── 일시정지 (Pause)
│   │   └── 설정 (Settings)
│   └── 게임 결과 (Result) ※ 게임오버 없음, 자발적 종료 시
├── 리더보드 (Leaderboard)
├── 상점 (Shop)
└── 설정 (Settings)
```

---

### 1.2 메인 메뉴 화면

게임 실행 시 가장 먼저 표시되는 화면이다. 심플하고 밝은 톤의 배경 위에 게임 타이틀과 핵심 버튼만 배치하여 직관적인 첫인상을 제공한다.

**구성 요소:**
- [x] 게임 타이틀 로고
- [x] 플레이 버튼 (가장 크고 눈에 띄는 CTA)
- [x] 이어하기 버튼 (저장된 게임이 있을 경우)
- [x] 리더보드 버튼
- [x] 상점 버튼
- [x] 설정 버튼
- [x] 사운드 ON/OFF 토글 (우측 상단)
- [x] 최고 점수 표시

## 2. 게임 보드 UI

### 2.1 헥사곤 그리드 레이아웃

게임 보드는 정육각형(Hexagon) 타일이 벌집 형태로 배열된 그리드를 사용한다. 오프셋 좌표계(Offset Coordinates)를 기반으로 구현한다.

**그리드 사양:**

| 항목 | 값 | 비고 |
|------|-----|------|
| 그리드 형태 | 벌집형 (Honeycomb) | Pointy-top 헥사곤 |
| 그리드 크기 | 가변 (5~7행) | 난이도에 따라 확장 가능 |
| 기본 행 수 | 9행 (다이아몬드형) | 3-4-5-6-7-6-5-4-3 배열 |
| 헥사곤 방향 | 꼭짓점이 위를 향함 (Pointy-top) | - |
| 좌표계 | 축 좌표 (Axial Coordinates) | q, r 좌표 |

**그리드 배열 다이어그램 (기본 다이아몬드형):**

```
행 인덱스     헥사곤 배열              셀 수

  Row 0          ⬡  ⬡  ⬡                3
  Row 1        ⬡  ⬡  ⬡  ⬡              4
  Row 2      ⬡  ⬡  ⬡  ⬡  ⬡             5
  Row 3    ⬡  ⬡  ⬡  ⬡  ⬡  ⬡            6
  Row 4  ⬡  ⬡  ⬡  ⬡  ⬡  ⬡  ⬡           7  (중앙)
  Row 5    ⬡  ⬡  ⬡  ⬡  ⬡  ⬡            6
  Row 6      ⬡  ⬡  ⬡  ⬡  ⬡             5
  Row 7        ⬡  ⬡  ⬡  ⬡              4
  Row 8          ⬡  ⬡  ⬡                3

  총 셀 수:  3+4+5+6+7+6+5+4+3 = 43셀
```

### 2.2 블록 크기 및 간격

**블록(헥사곤 타일) 치수:**

```
                ╱ ╲
              ╱     ╲         높이(H) = size * 2
            ╱    ●    ╲       너비(W) = sqrt(3) * size
          ╱      |      ╲
         │   size│       │    size = 기준 반지름
         │       │       │
          ╲      │      ╱
            ╲         ╱
              ╲     ╱
                ╲ ╱
```

| 항목 | 모바일 (세로) | 웹 (1920x1080) | 계산식 |
|------|-------------|---------------|--------|
| size (기준 반지름) | 28px | 40px | 화면 폭 기반 동적 계산 |
| 헥사곤 너비 (W) | 48.5px | 69.3px | sqrt(3) * size |
| 헥사곤 높이 (H) | 56px | 80px | size * 2 |
| 수평 간격 | 4px | 6px | 헥사곤 사이 간격 |
| 수직 간격 | 4px | 6px | 행 사이 간격 |
| 내부 여백 | 3px | 4px | 헥사곤 테두리 ~ 숫자 텍스트 |

**size 동적 계산 공식:**
```
availableWidth = 화면 폭 * 0.9  (좌우 여백 5%씩)
maxColumns = 7  (최대 열 수, 중앙 행 기준)
size = (availableWidth - (maxColumns - 1) * gap) / (maxColumns * sqrt(3))
```

## 4. 애니메이션 시스템

모든 애니메이션은 Unity의 DOTween 또는 내장 Animation 시스템을 활용하여 구현한다. 60fps를 기준으로 부드러운 동작을 보장한다.

### 4.1 블록 생성 애니메이션

새로운 블록이 보드에 처음 나타날 때 재생되는 애니메이션이다.

```
프레임 0       프레임 6        프레임 10      프레임 15
(0ms)          (100ms)        (167ms)        (250ms)

   ·              ⬡              ⬡              ⬡
               (30%)           (110%)          (100%)
              scale          scale           scale
              fade-in        overshoot       settle

시퀀스: Scale(0→1.1→1.0) + Fade(0→1)
```

| 속성 | 값 |
|------|-----|
| 총 시간 | 0.25초 (250ms) |
| 이징 | EaseOutBack (오버슈트 후 정착) |
| 시작 스케일 | 0.0 |
| 오버슈트 스케일 | 1.1 |
| 최종 스케일 | 1.0 |
| 투명도 | 0 -> 1 (EaseOutQuad) |
| 딜레이 | 블록별 0.03초 순차 딜레이 (파도 효과) |

---

### 4.2 블록 탭 선택 피드백

사용자가 블록을 탭했을 때 즉각적인 시각적 피드백을 제공한다.

```
일반 상태          탭 순간           선택 유지 상태

   ⬡               ⬡ ← 살짝 축소    ⬡ ← 글로우 효과
  (32)            (32)  0.95x       (32)  + 흰색 테두리
                   + 밝기 증가        + 미세 펄스
```

**첫 번째 블록 선택 (선택 시작):**

| 속성 | 값 |
|------|-----|
| 축소 | Scale 1.0 -> 0.95 -> 1.05 -> 1.0 (탄성 바운스) |
| 시간 | 0.15초 |
| 이징 | EaseOutElastic |
| 테두리 | 흰색 3px 글로우 테두리 추가 (fade in 0.1초) |
| 밝기 | 배경색 명도 +15% |
| 사운드 | "tap_select.wav" 재생 |
| 햅틱 | Light Impact (iOS) / 약한 진동 10ms (Android) |

**두 번째 블록 선택 (매칭 시도):**

| 조건 | 결과 |
|------|------|
| 같은 숫자 (매칭 성공) | 머지 애니메이션으로 전환 (4.3) |
| 다른 숫자 (매칭 실패) | 두 블록 모두 좌우 흔들림(shake) 0.2초 + 빨간 번쩍임 |
| 같은 블록 재탭 | 선택 해제, 테두리/글로우 fade out 0.1초 |

---

### 4.3 머지 애니메이션 (합쳐지는 모션)

두 개의 같은 숫자 블록이 합쳐지는 핵심 애니메이션이다. 가장 중요한 만족감 요소이므로 세밀하게 연출한다.

```
단계 1: 이동       단계 2: 합체        단계 3: 팽창       단계 4: 정착
(0~200ms)         (200~300ms)        (300~400ms)       (400~500ms)

 ⬡A    ⬡B         ⬡A                  ⬡AB               ⬡AB
(32)  (32)     →  (32)(32)  →      ( 64 )    →       ( 64 )
                   겹침               1.3x 확대          1.0x
                                     파티클 방출         안착

시퀀스:
1. 블록 B가 블록 A 위치로 이동 (EaseInQuad)
2. 두 블록이 합쳐지며 새 숫자 표시
3. 합쳐진 블록이 1.3배로 팽창 + 파티클 이펙트
4. 1.0배로 돌아오며 안착 (EaseOutBack)
```

| 속성 | 값 |
|------|-----|
| 총 시간 | 0.5초 (500ms) |
| 이동 이징 | EaseInQuad (가속 이동) |
| 팽창 이징 | EaseOutBack (오버슈트) |
| 파티클 | 합체 순간 원형 파티클 8~12개 방출 (블록 색상) |
| 파티클 수명 | 0.4초, 크기 감소 + 투명도 감소 |
| 숫자 전환 | 합체 순간 크로스페이드 (old fade out + new fade in, 0.1초) |
| 색상 전환 | 새 숫자의 색상으로 즉시 전환 |
| 사운드 | "merge.wav" 재생 (피치는 숫자 크기에 비례하여 상승) |
| 점수 팝업 | 합체 위치에서 "+점수" 텍스트 위로 떠오르며 페이드아웃 |

### 4.4 파도(웨이브) 애니메이션 (새 블록 등장)

머지 이후 빈 자리를 채우며 새로운 블록이 밀려 들어오는 애니메이션이다. 참고 게임의 핵심적인 시각 특징인 "파도처럼 밀려오는" 느낌을 구현한다.

```
단계 1: 빈 공간 감지    단계 2: 파도 진입      단계 3: 안착

 ⬡  ·  ⬡               ⬡  ⬡  ⬡              ⬡  ⬡  ⬡
 ·  ⬡  ·  ⬡        →   ⬡  ⬡  ⬡  ⬡     →    ⬡  ⬡  ⬡  ⬡
 ⬡  ·  ⬡  ·  ⬡         ⬡  ⬡  ⬡  ⬡  ⬡       ⬡  ⬡  ⬡  ⬡  ⬡

 (· = 빈 공간)          새 블록이 아래/옆에서    모든 블록 안착
                       순차적으로 밀려옴
```

**파도 방향:**

```
방향 패턴 (무작위 선택):

패턴 A: 아래→위        패턴 B: 좌→우         패턴 C: 외곽→중앙

     ← ← ←               → ⬡ ⬡                ← ← ←
   ← ← ← ←             → → ⬡ ⬡              → ⬡ ⬡ ←
  ← ← ← ← ←          → → → ⬡ ⬡            → → ⬡ ← ←
   ← ← ← ←            → → → → ⬡             → ⬡ ⬡ ←
     ← ← ←              → → → → →              ← ← ←
```

| 속성 | 값 |
|------|-----|
| 진입 방향 | 하단/좌측/우측 중 무작위 또는 순환 |
| 블록 간 딜레이 | 0.04초 (순차적으로 밀려오는 느낌) |
| 이동 거리 | 화면 바깥에서 목표 위치까지 |
| 이동 시간 | 0.3초/블록 |
| 이징 | EaseOutCubic (감속 도착) |
| 바운스 | 도착 시 미세 바운스 (EaseOutBounce, 0.1초) |
| 사운드 | "wave_whoosh.wav" (낮은 음에서 시작, 점점 높아짐) |

### 4.5 점수 팝업 애니메이션

머지 발생 시 획득 점수가 시각적으로 표시되는 효과이다.

```
시작 (0ms)     중간 (300ms)     종료 (800ms)

                 +250              +250
   +250         (위로 이동)      (페이드아웃)
 (나타남)       크기 유지        투명도 → 0
 scale 0→1     y축 -30px
```

| 속성 | 값 |
|------|-----|
| 시작 위치 | 머지 발생 블록 중심 |
| 이동 | Y축 위로 40px 이동 |
| 시간 | 0.8초 |
| 폰트 | Roboto Bold, 20px |
| 색상 | #FFD700 (금색), 외곽선 #000000 2px |
| 스케일 | 0 -> 1.2 -> 1.0 (EaseOutBack, 0.15초) |
| 페이드아웃 | 0.5초 시점부터 fade out (EaseInQuad) |
| 대형 점수 (1000+) | 크기 1.5배, 색상 빨강 #FF5722, 별 파티클 추가 |

### 4.6 콤보/연쇄 이펙트

연속 머지 시 콤보 카운터가 증가하며 화려한 이펙트가 재생된다.

```
콤보 x2             콤보 x3              콤보 x5+

  COMBO x2           COMBO x3            ✦ COMBO x5 ✦
  (일반)             (강조)              (최대 강조)
  흰색 텍스트        노란색 텍스트        무지개 텍스트
  1.0x               1.2x 스케일          1.5x 스케일
                     화면 미세 흔들림      화면 흔들림 + 화면 플래시
                                         배경 색상 펄스
```

| 콤보 | 배수 | 시각 효과 | 사운드 |
|------|------|---------|--------|
| x2 | 1.5배 | 텍스트 팝업 | "combo_2.wav" |
| x3 | 2.0배 | 텍스트 + 화면 미세 흔들림 | "combo_3.wav" (더 높은 피치) |
| x4 | 2.5배 | 텍스트 + 글로우 + 파티클 | "combo_4.wav" |
| x5+ | 3.0배 | 텍스트 + 화면 플래시 + 대형 파티클 | "combo_max.wav" (팡파르) |

**콤보 타이머:**
- 머지 후 2.0초 이내에 다음 머지를 하면 콤보 연결
- 타이머는 화면에 표시하지 않음 (직관적 플레이 유도)
- 콤보 중단 시 콤보 카운터 fade out (0.5초)

### 6.4 접근성 고려사항

다양한 사용자가 불편 없이 게임을 즐길 수 있도록 접근성을 고려한다.

**시각 접근성:**

| 항목 | 구현 방법 | 우선순위 |
|------|---------|---------|
| 색각 이상 (색맹) 대응 | 숫자 외에 블록 내부에 패턴/기호 추가 옵션 | 높음 |
| 고대비 모드 | 배경을 진한 색으로, 블록 테두리 강화 | 중간 |
| 텍스트 크기 | 큰 글자 모드 옵션 (폰트 크기 1.3배) | 중간 |
| 화면 읽기 | 블록 선택 시 숫자 음성 읽기 (TTS 연동, 선택적) | 낮음 |

**색각 이상 대응 패턴:**
```
일반 모드:        색각 이상 모드:

  ⬡           ⬡              각 숫자에 고유 패턴 부여
 (2)         (2 ·)            2: 점 1개

  ⬡           ⬡              4: 점 2개 (대각선)
 (4)         (4 :)

  ⬡           ⬡              8: 가로줄
 (8)         (8 ─)

  ⬡           ⬡              16: 세로줄
(16)        (16 │)

  ⬡           ⬡              32: 엑스
(32)        (32 ×)
```

**조작 접근성:**

| 항목 | 구현 방법 | 우선순위 |
|------|---------|---------|
| 터치 영역 확대 | 최소 터치 타겟 48x48dp (구글 가이드라인) | 높음 |
| 오탭 방지 | 블록 간 간격 최소 4px 확보 | 높음 |
| 시간 제한 없음 | 게임 특성상 시간 제한 없어 천천히 플레이 가능 | 기본 |
| 한 손 조작 | 세로 모드에서 한 손 탭만으로 모든 조작 가능 | 높음 |
| 실수 복구 | Undo 기능으로 잘못된 머지 취소 가능 | 중간 |

**인지 접근성:**

| 항목 | 구현 방법 | 우선순위 |
|------|---------|---------|
| 간단한 규칙 | 같은 숫자 탭 = 합치기, 단 하나의 메커니즘 | 기본 |
| 게임오버 없음 | 스트레스 없는 플레이 환경 | 기본 |
| 진행 저장 | 자동 저장으로 언제든 중단/재개 가능 | 높음 |
| 힌트 시스템 | 막힐 때 광고 시청으로 힌트 획득 | 중간 |
| 피로 방지 | 30분 이상 플레이 시 "휴식 알림" 표시 (선택적) | 낮음 |

**접근성 구현 체크리스트:**
- [ ] 색각 이상 모드 옵션 추가 (설정 화면)
- [ ] 블록 내부 패턴 에셋 제작 (6종 이상)
- [ ] 고대비 모드 옵션 추가
- [ ] 큰 글자 모드 옵션 추가
- [ ] 터치 타겟 최소 48x48dp 확보
- [ ] 블록 간 간격 검증
- [ ] Undo 기능 구현
- [ ] 자동 저장 시스템
- [ ] 휴식 알림 시스템 (선택적)

| 분류 | 해상도 범위 | 보드 크기 조절 | UI 스케일 | 비고 |
|------|-----------|-------------|----------|------|

### 7.3 터치 vs 마우스 입력 처리

웹과 모바일에서 각각 다른 입력 방식에 대응하는 처리 방법이다.

**입력 이벤트 매핑:**

| 동작 | 터치 (모바일) | 마우스 (웹) | Unity Input |
|------|-------------|-----------|-------------|
| 블록 선택 | Tap | Left Click | Pointer Down + Pointer Up |
| 블록 호버 | N/A (없음) | Mouse Hover | Pointer Enter / Exit |
| UI 버튼 | Tap | Left Click | Pointer Click |
| 스크롤 | Swipe | Scroll Wheel / Drag | Scroll Rect |
| 슬라이더 | Drag | Drag | Slider Component |

**터치 전용 처리 (모바일):**

```
┌─────────────────────────────────────┐
│          터치 입력 처리 로직          │
├─────────────────────────────────────┤
│                                     │
│  Touch Down                         │
│       │                             │
│       ├── 블록 영역?                 │
│       │    ├── YES → 블록 선택 처리   │
│       │    │        (즉시 피드백)     │
│       │    └── NO  → 무시            │
│       │                             │
│  Touch Up                           │
│       │                             │
│       ├── 같은 블록 위?              │
│       │    ├── YES → 탭 확정         │
│       │    └── NO  → 탭 취소         │
│       │                             │
│  이동 거리 > 10px?                   │
│       ├── YES → 드래그로 판정, 탭 취소│
│       └── NO  → 탭으로 판정          │
│                                     │
└─────────────────────────────────────┘
```
